## Conversion Guidelines

### General Requirements
1. Follow Unreal Engine coding standards
2. Use appropriate UCLASS, UPROPERTY, and UFUNCTION macros
3. Include proper header guards and forward declarations
4. Implement proper error handling and validation
5. Maintain Blueprint's original functionality exactly

### Class Structure
- Preserve the original class hierarchy
- Maintain all Blueprint variables as UPROPERTY members
- Convert Blueprint functions to UFUNCTION methods
- Keep component relationships intact

### Variable Conversion
- Blueprint variables → UPROPERTY member variables
- Maintain original access specifiers (public/private/protected)
- Preserve replication settings where applicable
- Include proper categories and metadata

### Function Implementation
- Convert graph logic to C++ implementation
- Maintain execution flow from Blueprint graphs
- Handle async operations appropriately
- Preserve pure function designations

### Event Handling
- Convert Blueprint events to appropriate C++ event handlers
- Maintain event binding relationships
- Preserve multicast delegate connections

### Component Management
- Convert component hierarchy correctly
- Initialize components in constructor
- Set up component attachments properly

### Code Style
- Use UE coding conventions (prefix classes with A, U, F, etc.)
- Include appropriate log statements for debugging
- Add minimal but useful comments for complex logic
- Group related functionality together
